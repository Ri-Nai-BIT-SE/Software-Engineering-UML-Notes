# 需求规格说明文档

## 作用

- 更好的**传递**软件系统的**需求信息**和**解决方案**给所有的开发者作为**合同协议**的**重要部分**
- 作为项目开发活动的一个**重要依据**
- 发现和**减少**可能的需求**错误**，减少项目的返工，降低项目的工作量
- 拓展人们的**知识记忆能力**
- 作为有效的**智力资产**

## 忽视的原因

- 交流途径
- 时间压力
- 迭代式开发
- 敏捷

## 作者

- 项目管理者
  - 组织安排、提供条件
- 需求工程师
  - 负责人、主导人
- 文档写作人员
  - 有时会采用，节省需求工程师的时间
- 涉众（用户）
  - 验证人
